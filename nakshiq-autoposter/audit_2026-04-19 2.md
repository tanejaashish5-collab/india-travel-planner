   - **Git push conflicts.** Ashish was pushing feature work aggressively throughout the morning — commits at AEST 11:01, 12:15, 12:53, 13:23, 13:43, 14:12, 14:18, 14:39, 15:09–15:31. Any concurrent GH Actions run would have failed `git push` on the workflow's final commit step, losing the log + state diff. This matches the **fix at AEST 17:27 `fix: add retry-with-rebase to autoposter git push step`** — confirming push rejection has been biting.

   - **GH Actions scheduled-run delay.** GitHub's hosted crons often drift 5–30 min, occasionally drop entirely during platform load — less likely given 5+ consecutive slots are missing.

2. **YT Short slot 2 (AEST 12:00) — cannot confirm publication.** No log, no commit. State.json `m8EAd_yt_short` and `4QL0i_yt_short` both still show `2026-04-18` (yesterday's slot 3). A full 24-hour gap in YouTube Shorts cadence is the real risk.

4. **Audit task schedule mismatch.** The schedule in the task file is off by ~5 hours vs. the actual workflow. Audit framework for future days needs to be updated to avoid false-positive gap reporting.

5. **Stale lock at 05:27 UTC Apr 19 (age=89183s ≈ 24.8h).** Log line `Stale lock (age=89183s) — overriding.` suggests an earlier run crashed hard enough to leave the lockfile behind.
